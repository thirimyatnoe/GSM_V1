Global Setting Setup


  • System Utilities အောက်မှ Global Setting Form ကိုဖွင့်ပါ။
  • Logo ထဲတွင် မိမိဆိုင်၏ Logo အမှတ်တံဆိပ်ကို (.png) Format ဖြင့် ထည့်သွင်းသတ်မှတ်နိုင်မည်။ Stock သတ်မှတ်ရာတွင် Default Photo အနေဖြင့် ပြပေးမည် ဖြစ်သည်။
  • Gem Price သတ်မှတ်ရာတွင် မိမိဆိုင်၌ အသုံးပြုလိုသော ယူနစ်ကို ထည့်သွင်းပေးရမည်။ Normal – RBP or Ct မိမိ ထည့်သွင်းတာသည့် Unit အတိုင်း Price ကိုတွက်ချက်ပေးမည်။ရတီ - RBP ရတီ ဘီ စိတ် Unit ဖြင့်သာ Price ကိုတွက်ချက်ပေးမည်။ကာရက် - ကာရက်(Ct) Unit ဖြင့်သာ Price ကိုတွက်ချက်ပေးမည်။
  • Discount ခွင့်ပြုငွေ Administrator Level မှလွှဲပြီး တစ်ခြား User Levels များအား အများဆုံးပေးလျော့စေလိုသည့် ပမာဏ ကို ထည့်သွင်းပေးရမည် ဖြစ်သည်။ အရောင်းဘောင်ချာ ဖွင့်ရာတွင် ရိုးရိုး User Levels များသည် ယင်းပမာဏထက် ကျော်၍ လျော့ပေးနိုင်မည် မဟုတ်ပါ။
  • ခြားနားငွေ ၀◌ယ်‌ေဈး လဲစေ◌ျး ဆိုသည်မှာ မိမိဆိုင်ရှိ ပေါက်စေ◌ျးနှင့် ပြန်လည်၀◌ယ်ယူသောစေ◌ျး၊ လဲလှယ်သောစေ◌ျး ခြားနားလိုသော ပမာဏကိုထည့်သွင်းနိုင် မည့် နေရာဖြစ်သည်။ (*မိမိဆိုင်သည် ကာလပေါက်စေ◌ျး အတိုင်းပြန်လည် ၀◌ယ်ယူ လဲလှယ် ပေးသောဆိုင် ဖြစ်ပါက ထည့်သွင်းစရာ မလိုပါ။)
  • Decimal Format မိမိ ဆိုင်တွင် Y (ရွေး)နေရာ၌ အသုံးပြုလိုသော ဒသမ အလုံးရေ ကို ထည့်သွင်းနိုင်မည် ဖြစ်သည်။(ဥပမာ ၁ ဟု သတ်မှတ်ခဲ့ပါ က Software ထဲရှိ ရွေးနေရာ အားလုံးတွင် ဒသမတစ်နေရာသာ ယူမည်ဖြစ်ပြီး ၂ ဟု သတ်မှတ်ခဲ့ပါ က နှစ်နေရာယူတွက်သွားမည်ဖြစ်သည်။ Y – 4.3 or 4.25 )
  • Share Path – Software ၏ Database Backup File သိမ်းဆည်းမည့်လမ်းကြောင်းကို ထည့်သွင်းသတ်မှတ်ပေးရမည်။
  • QR Code မိမိဆိုင်၏ အရောင်းဘောင်ချာများတွင် QRCode အတွင်း ထည့်သွင်းသတ်မှတ်လိုသည့် information များကို ရေးသားနိုင်သည်။QRCode သုံးမည်ဆိုပါက မိမိဆိုင်၏ Voucher Template တွင် QRCode ကို Global Gold Service Team မှ ထည့်သွင်းပေးထားရန်လိုမည် ဖြစ်သည်။
  • Exactly Calculate Price Setting ကို အမှန်ခြစ်ထားပါက ရွှေပေါက်စေ◌ျးသတ်မှတ်ရာ၌ အတိအကျတွက်ချက်ပေးမည်ဖြစ်သည်။ Exactly Calculate Price Setting ကို အမှန်ခြစ်ဖျောက်ထားပါက အစွန်းထွက်များအား ၅ ဖြစ် ၅ ကျော် အတိုး အလျော့လုပ်ပေးပြီး ယူမည်ဖြစ်သည်။
  • Sale Return အရောင်းဘောင်ခါများအား ပြန်လည် ၀◌ယ်ယူရာတွင် IsAllowToControlSaleReturn(Sale Return Voucher များအား ထိန်းချုပ်ခွင့် ပြုမည်) Setting ကို on ထားပါက ရိုးရိုး User levels များသည် ပြန်၀◌ယ်လဲ ဘောင်ချာများအား Update / Delete ပြုလုပ်နိုင်မည်ဖြစ်သည်။Offထားပါက System Admin သာလျှင် Update / Delete ပြုလုပ်နိုင်မည်ဖြစ်သည်။
  • IsCash Setting ကို on ထားပါက အရောင်းဘောင်ချာဖွင့်ရာတွင် ပေးငွေသည် ရိုက်ထည့်စရာ လိုတော့ပဲ Auto ကျပေးမည်ဖြစ်သည်။Stock Setup Form တွင် ၀ယ်ရင်းစျေးကို ထည့်စရာမလိုပဲအချက်အလက်မျာထည့်သွင်းပြီး Barcode ထုတ်နိုင်ပါသည်။ CheckBox မ on ထားပဲ Stock သွင်းပြီး Barcode ထုတ်လျှင် “Please Fill Purchase Price” ဟူသော Message Box ကျလာမည်ဖြစ်သည်။
  • အရောင်းဘောင်ချာများအား IsAllowToControlSale (SaleVoucher များအား ထိန်းချုပ်ခွင့် ပြုမည်)Setting ကို on ထားပါက ရိုးရိုး User levels များသည် ဘောင်ချာများအား Update / Delete ပြုလုပ်နိုင်မည်ဖြစ်သည်။Offထားပါက System Admin သာလျှင် Update / Delete ပြုလုပ်နိုင်မည်ဖြစ်သည်။
  • IsAllowItemToUpdate(အရောင်းဘောင်ချာဖွင့်ရာတွင် အထည်ချိန် အလျော့တွက်ကို ထိန်းချုပ်ခွင့်ပြုမည်။) Setting ကို on ထားပါက ရိုးရိုး User levels များသည် ဘောင်ချာဖွင့်ရာတွင် အထည်၏ အလေးချိန်များအား Update / Delete ပြုလုပ်နိုင်မည်ဖြစ်သည်။Offထားပါက System Admin သာလျှင် Update / Delete ပြုလုပ်နိုင်မည်ဖြစ်သည်။
  • AllowEditWeight ---g ဘောင်ချာဖွင့်ရာတွင် ရိုးရိုး User levels များအား အထည်၏ အလေးချိန်များကို ပြုပြင်ရာ၌ အများဆုံးပေးပြင်စေလိုသော ပမာဏကို ထည့်သွင်း၍ ကန့်သတ်နိုင်မည် ဖြစ်သည်။
  • Speed Entry Setting ကို on ထားပါက Stock Setup ပြုလုပ်ရာတွင် အမျိုးအစားအမျိုးအမည် အလျော့တွက် တူရာများ ဆက်တိုက်ထည့်သွင်းမည်ဆိုပါ က ပြန်လည်ရွေးချယ်စရာမလိုပဲ လျင်မြန်စွာ ပစ္စည်းထည့်သွင်းနိုင်မည် ဖြစ်သည်။
  • IsAllowItemToControlStock(သတ်မှတ်ထားပြီးသော ပစ္စည်းများအား ပြန်လည် ထိန်းချုပ်ခွင့်ပြုမည်။) Setting ကို on ထားပါက ရိုးရိုး User levels များသည် ဘောင်ချာဖွင့်ရာတွင် အထည်၏ အလေးချိန်များအား Update / Delete ပြုလုပ်နိုင်မည်ဖြစ်သည်။Offထားပါက System Admin သာလျှင် Update / Delete ပြုလုပ်နိုင်မည်ဖြစ်သည်။
  • Resuse Barcode Setting ကို on ထားပါက ယင်းပစ္စည်းသည် ကုတ်ပျက်(IsClosed) ဖြစ်သွားပါက ထိုဘားကုတ်နံပတ်ကို ပြန်လည်အသုံးပြုနိုင်မည် ဖြစ်သည်။
  • Interest Rate အပေါင်အသုံးပြုရာတွင် မိမိသတ်မှတ်လိုသော အတိုးနှုန်းကို သတ်မှတ်ထားနိုင်မည်ဖြစ်သည်။
  • Interest Period အပေါင်အသုံးပြုရာတွင် မိမိသတ်မှတ်လိုသော အပေါင်ဆုံးကာလ (month) ကို အသေသတ်မှတ်ထားနိုင်မည်ဖြစ်သည်။ UseSettingInterestPeriod အပေါင်ဆုံးကာလကို အသုံးပြုမည် ဟူသော Setting ကို on ထားရမည် ဖြစ်သည်။ UseSettingInterestPeriod အပေါင်ဆုံးကာလကို အသုံးမပြုပါက အပေါင်ဘောင်ချာ ဆွဲရာတွင် အပေါင်ဆုံးကာလကို မိမိစိတ်ကြိုက်သတ်မှတ်နိုင်မည် ဖြစ်သည်။
  • ရက်လွန်နေ့ အသုံးပြုမည် ကို အမှန်ခြစ်ထားပါက ရက်လွန်နေ့ထဲတွင် သတ်မှတ်ထားသောရက်နှင့် ရက်ထက် ကျော်လွန်ပါက တစ်လစာ (ရက် ၃၀) အပြည့်ယူသွားမည်ဖြစ်ပြီး သတ်မှတ်ထားသောရက်ထက်နည်းပါက လ၀◌က်စာ (၁၅ ရက်) ရက်လွန်ကြေးအဖြစ်ကောက်ခံသွားမည်ဖြစ်သည်။
  • Transfer=>UseHoToBranchဆိုင်ခွဲများမှရုံးချုပ်သို့ပစ္စည်းပေးပို့မည် သတ်မှတ်လိုက်ခြင်းဖြစ်သည်။ ဆိုင်ခွဲများမှာ ပစ္စည်းထည့်သွင်းမည် ဆိုပါက ထို Setting ကို အမှန်ခြစ်ထားရန်မလိုပေ။
  • Using Master From Ho=>ရုံးချုပ်မှဆိုင်ခွဲများရှိ master data များကို ထည့်သွင်းမည်ဆိုပါက ထိုSetting ကို on ပေးရမည် ဖြစ်ပြီး ဆိုင်ခွဲများမှာ master data များကို ထည့်သွင်းမည် ဆိုပါက ထို Setting ကို အမှန်ခြစ်ထားရန်မလိုပေ။
  • (မှတ်ချက် ။ Global Setting သည် Software တစ်ခုလုံးကို ထိန်းချုပ်ပေးသည့် Setting များဖြစ်သောကြောင့် တစ်ခုခု ပြင်ဆင်ပြောင်းလဲသတ်မှတ်ပြီးပါက Exit တစ်ခါထွက်ပြီး ပြန်၀◌င်မှသာ ပြောင်းလဲသွားမည် ဖြစ်သည်။)
  • Data များဖြည့်စွက်ပြီးပါက Save Button ကိုနှိပ်၍ သိမ်းဆည်းနိုင်ပါသည်။
